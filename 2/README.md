## üéØ Design Principles

### Single Responsibility Principle
Each folder has ONE clear purpose. Each file within a folder addresses ONE specific concern.

### Separation of Concerns
- **Configuration** (`.env`, config files) ‚â† **Instructions** (`CLAUDE.md`) ‚â† **Knowledge** (`memory-bank/`)
- **Static knowledge** (workflows, templates) ‚â† **Dynamic cache** (API data) ‚â† **Ephemeral context** (session notes)

### Discoverability
- Numbered folders indicate processing order
- Clear naming conventions
- README.md in each folder serves as a catalog

### Immutability vs. Mutability
- Folders 01-04, 07-09, ticket-administration: Committed to git, version-controlled knowledge
- Folder 05: Cached data, regenerated as needed, gitignored
- Folder 06: Mixed - documentation files (*.md) committed, working folders (LOCAL/, JIRA/) gitignored

## üìñ Usage Guidelines for Claude Code

### Priority Order
1. **Workflows first**: Always start with `01-workflows/` for task procedures
2. **Templates second**: Use `02-templates/` for formatting
3. **Architecture third**: Reference `03-component-architecture/` for codebase understanding
4. **Standards fourth**: Apply `04-standards/` for quality
5. **Cache/Context last**: Check `05-cache/` and `06-context/` for runtime data

### Context Scoping
- Only read files relevant to the current task
- Avoid reading entire folders unless necessary
- Use folder READMEs as indexes to find specific files

### Cache Usage
- Check `05-cache/` before making redundant API calls
- If cache is stale or missing, regenerate it
- Cache files use JSON format for machine-readability

### Context Management
- Use `06-context/LOCAL/localN/` for drafting ticket requirements before ticket system creation
- Use `06-context/JIRA/xxx-XXX/` for working on fetched tickets (adjust naming for your ticket system)
- Documentation files (`06-context/*.md`) are committed as source of truth
- Working folders (`LOCAL/`, `JIRA/`) are gitignored for privacy
- After completion, archive to `ticket-administration/` (uppercase naming)
- See `06-context/README.md` for complete structure and workflows
